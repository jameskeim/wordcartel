# Effort H17 — `wordcartel-core` public-API doc-coverage sweep + `missing_docs` gate

**Status:** SPEC (2026-07-11). Backlog item **H17** (`docs/ux-backlog.md`, Theme H). Last of the
pre-Effort-P cleanup sequence (A ✅ → B ✅ → **H17**). Scope decided with the human 2026-07-11:
**Scope A — core-only, gated** (of the three options core-only / both-crates / core+shell-tightening).

## Goal

Doc-comment every undocumented public item in **`wordcartel-core`** (the pure edit kernel — the
genuine reusable/public API), then land **`#![warn(missing_docs)]`** on the crate root as a durable
gate so the coverage cannot silently regress. Effort P exposes this surface to plugins; it should be
documented and stay documented.

## Scope

**In scope:** the **237** undocumented public items in `wordcartel-core`, enumerated by the compiler
(Phase 0 worklist below), plus the crate-root gate attribute. (The `lib.rs`=1 hit *is* the undocumented
`pub mod block_tree;` declaration — resolved by giving `block_tree.rs` its missing `//!` module doc, so
it is not a separate item beyond the 237.) The 237 is the **`cargo test --no-run` surface** (234 under a plain `cargo build`, **plus 3**
`cfg(any(test, fuzzing))`-gated public fields in `test_support::EditOp`) — because the repo's
warning-free gate covers `cargo test --no-run`, so `#![warn(missing_docs)]` fires on the `cfg(test)`
`test_support` module too. Scoping to the build-only 234 would leave the gate red on the test compile.

**Out of scope (explicit):**
- The **`wordcartel` shell crate** (660 `missing_docs` hits). Most are `pub`-for-cross-module, not a
  consumed API; documenting them doesn't serve the plugin goal. Deferred — best handled later as a
  `pub → pub(crate)` visibility-tightening pass on its own, not smuggled into a doc sweep.
- The **`repar` path-dep** (44 hits) — not our code.
- **Any visibility change in core.** H17 *documents* the existing public surface; it does not add,
  remove, or narrow `pub`. (See anti-cheat below — the lint must not be silenced by hiding items.)

## Policy decisions (the resolved forks — binding on plan + implementers)

1. **Gate = `#![warn(missing_docs)]`, not `deny`.** Placed in `wordcartel-core/src/lib.rs` alongside
   the existing crate-level attr block (`#![forbid(unsafe_code)]`, the H7 `#![deny(clippy::cast_*)]`).
   Rationale: teeth already come from the **warning-free-build GATE** (CLAUDE.md Rust conventions —
   `cargo build`/`cargo test --no-run` must be warning-free for touched crates), so `warn` is
   effectively enforced at merge without `deny`'s harsher local-iteration friction. Consistent with
   the earlier decision recorded for this item.
2. **Doc-content standard = house style** (CLAUDE.md "Docs"): doc-comment every public item; document
   params/returns/errors where they exist. **Substance over stubs** — a terse-but-real sentence for a
   field/variant, not `/// The foo.` echoing the name. Fields and enum variants get a short phrase;
   functions get purpose + params/returns/errors.
3. **`# Examples` only on non-obvious public *functions/methods*.** Per CLAUDE.md ("a runnable
   `# Examples` block for non-obvious public functions") — NOT on fields, enum variants, type aliases,
   or trivial accessors/getters. "Non-obvious" is the reviewer's judgment. Any example added is a
   **runnable doctest** (`cargo test` compiles/runs it) — keep them minimal and correct.
4. **Anti-cheat (reviewer-enforced).** The lint can be silenced two illegitimate ways: an empty stub
   doc, or reducing an item's visibility to hide it. **Both are defects.** Default is to *document in
   place*; no `pub → pub(crate)` to dodge the lint (and per scope, no visibility change at all).
5. **Module docs count.** `block_tree.rs` lacks a `//!` inner doc (the sole `lib.rs`-level hit) — add
   one. All other core modules already carry `//!` docs.
6. **Fuzz-exposed public items — convention, not new gate machinery.** The durable gate compiles the
   `build` + `test --no-run` + `clippy` surfaces, but **not** `--cfg fuzzing`. So the standing
   **convention** (which both existing fuzz-exposed items already follow — `test_support`,
   `block_tree::incremental_equals_full`) is: any fuzz-exposed public item is gated
   `#[cfg(any(test, fuzzing))]`, never `#[cfg(fuzzing)]` alone — which brings it under the
   `cargo test --no-run` compile the gate already covers. A hypothetical `#[cfg(fuzzing)]`-only public
   item would escape the gate; that is **deliberately out of scope** (near-zero blast radius — a
   fuzz-only surface, never shipped or plugin-facing; calibrating guard strength to blast radius per
   the H7 stance). Adding a `--cfg fuzzing` doc-check to CI is the stronger alternative if ever wanted;
   H17 chooses the convention. *(Codex round-2 finding, resolved as a scoping decision.)*

## Verification & gates

- **Intrinsic (the point of the effort):** after the sweep + attribute, both `cargo build -p
  wordcartel-core` **and** `cargo test -p wordcartel-core --no-run` are **warning-free** (the
  `missing_docs` worklist is empty on the full test-compile surface — this is the gate that catches the
  3 `test_support` fields), `cargo test` is green (including any new doctests), and `cargo clippy
  --workspace --all-targets` is clean.
- **No gate collision:** `module_budgets` bounds only shell hubs (`app.rs`/`render.rs`/`timers.rs`) —
  no core file; `clippy::too_many_lines` counts function-body lines — doc comments are outside bodies.
  So H17 cannot trip either anti-regrowth gate.
- **Review layers (calibrated — lighter than Effort A):** per-task reviewer checks **doc quality** (no
  stubs, no hide-to-dodge, params/errors present, examples compile) — the one place judgment is needed
  since the compiler enforces *completeness* but not *substance*. **ONE final gate: Codex pre-merge**
  (spot-check doc accuracy against the code; confirm no visibility was narrowed to silence the lint;
  confirm examples are correct). **No Fable whole-branch** — nothing cross-cutting; the compiler
  enforces completeness, so there is no cross-task invariant for Fable to synthesize.

## Command-surface-contract conformance

**N/A — does not touch the command surface.** H17 adds only doc comments and one crate-level lint

removed, or changed. (Stated explicitly per CLAUDE.md's command-surface-contract requirement.)

## Risks / notes

- **`theme.rs` volume (84):** exhaustive `SemanticElement` variants and chrome-face fields dominate.
  Terse, accurate one-liners — resist inventing semantics; describe what each face/field *is*.
- **Doctests become tests:** an incorrect `# Examples` block fails `cargo test`, not just review — a
  feature (it keeps docs honest), but implementers must actually compile examples.
- **`diagnostics.rs`** data types survived Effort A's Harper-invocation deletion; document them as the
  provider-neutral diagnostic vocabulary they now are.
